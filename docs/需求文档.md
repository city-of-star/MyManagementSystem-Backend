# 高校在线实习管理平台 — 需求说明

> 文档版本：v0.2  
> 说明：面向本科毕设的实现范围；与《接口文档》《新增sql》配套使用。

---

## 1. 项目背景与目标

高校组织学生赴合作企业开展实习，涉及企业信息、岗位发布、学生报名、过程管理（周志）、校企评价与成绩归档等环节。传统线下表格效率低、难追溯。本系统目标为：

- 提供统一的在线实习流程管理与材料归档；
- 采用微服务架构，配套统一身份认证、文件存储、操作留痕与定时任务等能力；
- 规模以满足毕设演示与论文撰写为主，不追求高并发与复杂审批流。

---

## 2. 建设原则

| 原则 | 说明 |
|------|------|
| 基础能力统一 | 用户、角色、组织机构、菜单、字典、文件、登录与安全审计等由平台公共能力统一提供 |
| 业务内聚 | 实习领域的数据与接口集中在实习业务服务中，边界清晰 |
| 适度简化 | 企业导师可弱化（仅评语/确认）；电子签章改为「确认 + 操作日志」 |
| 数据范围 | 通过角色与组织机构范围控制教师、管理员可见数据（实现可逐步加强） |

---

## 3. 角色与职责

| 角色 | 主要职责 |
|------|----------|
| 学生 | 维护个人实习相关信息、浏览岗位、报名/撤销（在规则允许内）、提交周志与实习材料、查看评价与成绩 |
| 校内指导教师 | 查看所指导学生、审核报名（若流程需要）、批阅周志、填写校内鉴定与评分 |
| 企业导师（可选） | 查看绑定学生的实习情况、填写企业侧评语/评分（可由校内教师代录以简化实现） |
| 学院/教务管理员 | 维护实习批次、审核合作企业与岗位、监督进度、统计与导出 |
| 系统管理员 | 用户与账户、角色权限、菜单与系统参数等全局管理 |

实习业务中的师生均对应平台用户主键；报名、周志、评价等表通过用户 ID 与平台账户关联。

## 4. 功能需求

### 4.1 基础数据与批次

- **实习批次**：学年、学期或自定义批次名称；报名起止时间；是否允许学生自主报名等开关（可用配置项或批次字段表达）。
- **合作企业**：企业名称、统一社会信用代码（可选）、联系人、电话、地址、简介；状态：待审核 / 已通过 / 已停用。
- **实习岗位**：关联企业与批次；岗位名称、人数上限、岗位要求说明、实习起止时间；状态：草稿 / 已发布 / 已结束。

### 4.2 报名与分配

- 学生浏览「已发布」岗位并提交报名，产生报名记录。
- 报名状态至少包含：待审核、已通过、已拒绝、学生已撤销。
- 支持管理员或校内教师将学生**分配**至某岗位（可与「报名」二选一或并存，实现阶段可先做一种）。

### 4.3 实习过程

- **周志**：按周或双周提交；富文本内容；可关联统一文件上传能力所生成的附件标识（如业务类型 + 业务主键与周志记录对应）。
- **材料**：实习协议、安全告知等以附件形式提交；列表展示审核状态（可选：待交/已交/已审）。

### 4.4 考核与归档

- **校内评价**：评分（数值）、文字鉴定。
- **企业评价**：评分与评语（可简化）。
- **综合成绩**：按简单规则合成（如校内 60% + 企业 40%），规则可配置或写在服务端（论文中可说明可扩展）。
- **导出**：支持按批次/学院导出 Excel 或鉴定表模板（实现优先级次于核心流程）。

### 4.5 统计与查询（管理端）

- 按批次统计：报名人数、通过人数、在实习人数、已完成人数。
- 按学院/专业统计：依赖学生与组织机构（院系、班级等）的关联数据。

### 4.6 非功能需求（毕设级）

- **安全**：接口经网关鉴权；敏感操作写入操作日志，可追溯。
- **可用性**：Web 端为主；移动端非必须。
- **性能**：演示环境单实例即可；论文中可说明后续水平扩展方式。

## 5. 业务状态（建议）

**报名记录 `application_status`**

- `PENDING` 待审核  
- `APPROVED` 已通过  
- `REJECTED` 已拒绝  
- `CANCELLED` 学生撤销  
- `IN_PROGRESS` 实习进行中（可与「已通过」合并，视实现而定）  
- `COMPLETED` 已结束  

**周志 `weekly_log_status`（可选）**

- `SUBMITTED` 已提交  
- `APPROVED` 教师已通过  
- `REJECTED` 退回修改  

---

## 6. 逻辑架构（能力划分）

| 能力 | 说明 |
|------|------|
| 统一认证与访问控制 | 登录、令牌校验、用户/角色/权限、菜单 |
| 文件服务 | 附件上传、存储与访问 |
| 任务调度 | 如批次截止后自动关闭岗位、定时统计等 |
| 实习业务 | 企业、批次、岗位、报名、周志、评价与成绩 |

各模块在部署上可对应独立微服务，具体拆分与注册发现方式在总体设计或论文中阐述。

---

## 7. 实现顺序建议

1. 在业务库中执行 `新增sql.sql`，创建实习相关表。  
2. 按《接口文档》实现实习业务服务的接口与持久层。  
3. 完成 Web 前端菜单与页面，经 API 网关访问实习服务。
