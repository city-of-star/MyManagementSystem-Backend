# 环境与运维资源

## 1. `mysql` 目录

- 初始化脚本：
  - `mysql/init_mms_dev_core.sql`
  - `mysql/init_mms_prod_core.sql`
- 两套脚本结构一致，核心是库名不同（`mms_dev_core` vs `mms_prod_core`）。
- 数据模型覆盖 RBAC 与基础能力：
  - 用户/角色/权限：`system_user`、`system_role`、`system_permission`
  - 关联表：`system_user_role`、`system_role_permission`
  - 组织与岗位：`system_dept`、`system_post`、`system_user_dept`、`system_user_post`
  - 其他基础表：`system_config`、`system_dict_type`、`system_dict_data`、`system_attachment` 等
- 表设计普遍采用：
  - 逻辑删除字段：`deleted`
  - 审计字段：`create_by/create_time/update_by/update_time`

## 2. `nacos` 目录

## 2.1 配置分层

- 公共配置：
  - `public-DEV.yaml`
  - `mysql-DEV.yaml`
  - `redis-DEV.yaml`
  - `jwt-DEV.yaml`
  - `secret-DEV.yaml`
  - `threadpool-DEV.yaml`
  - `log-DEV.yaml`
  - `whitelist-DEV.yaml`
  - `websocket-DEV.yaml`
- 服务配置：
  - `gateway-DEV.yaml`
  - `usercenter-DEV.yaml`
  - `base-DEV.yaml`
  - `job-DEV.yaml`

## 2.2 关键约束

- 所有 `YOUR_*` 为占位符，真实敏感值应在运行环境 Nacos 中注入，禁止写入仓库。
- 网关签名密钥分离：
  - 网关：`gateway-DEV.yaml` 使用 `gateway.signature.private-key`
  - 下游服务：`secret-DEV.yaml` 使用 `gateway.signature.public-key`
- 服务端口默认：
  - `usercenter` = 5090
  - `base` = 5091
  - `gateway` = 5092
  - `job` = 5093
- 网关路由前缀默认 `/api/{service}/**`，并通过 `StripPrefix=2` 转发。
- 白名单由 `whitelist-DEV.yaml` 统一维护，网关与服务侧均依赖白名单策略。

## 3. `script` 目录

## 3.1 后端脚本

- `mms-后端服务重启脚本.sh`
  - 停旧进程 -> 归档新 jar 到 `history` -> 更新 `current.jar` 软链接 -> 启动 -> tail 最新日志
  - 使用 `run.pid` 管理进程
- `mms-后端切换版本.sh`
  - 从 `history/*.jar` 交互选择版本 -> 切换 `current.jar` -> 启动并跟日志
- `mms-view-log.sh`
  - 跟踪 `./logs/{模块名}` 下最新日志文件

## 3.2 前端脚本

- `mms-前端发布脚本.sh`
  - 接收 `dist` 目录或 `dist.zip` -> 生成版本目录 -> 切换 `current` 软链接 -> 清理历史版本
  - 尝试继承上一版本 `config/index.js`
- `mms-前端切换版本.sh`
  - 从 `history` 交互选择版本并切换 `current` 软链接

## 4. 变更与排障建议

- 修改 Nacos 配置时，优先区分「公共配置」与「服务配置」影响面。
- 涉及鉴权故障时，联动检查：
  - `jwt-DEV.yaml`
  - `secret-DEV.yaml`
  - `gateway-DEV.yaml`
  - `whitelist-DEV.yaml`
- 涉及服务起停故障时，优先检查：
  - `run.pid` 与实际进程一致性
  - `current.jar` 软链接目标
  - `logs/{模块名}` 最新日志
