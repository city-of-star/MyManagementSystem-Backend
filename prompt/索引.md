# MMS 后端 AI 索引

## 0. 执行协议（必须遵守）

1. 优先读取本文件。
2. 根据「2. 任务路由」选择需要读取的文档。
3. 读取必要文档后，再根据 `任务导航/任务入口索引.md` 定位代码入口。
4. 文档与代码行为不一致时，以代码为准，并回补文档。

## 1. 文档地图

- `索引.md`：全局路由与执行协议。
- `架构/项目总览.md`：架构基线、模块职责、分层模式。
- `架构/系统能力清单.md`：现有通用能力与业务能力盘点（避免重复造轮子）。
- `安全/鉴权链路.md`：网关鉴权、请求头透传、服务验签、权限校验链路。
- `架构/开发约定.md`：实现约束与一致性规则。
- `任务导航/任务入口索引.md`：任务到代码路径的映射。
- `运行运维/环境与运维资源.md`：`mysql`、`nacos`、`script` 的运行与运维约束。
- `维护/维护规范.md`：本知识库的维护规范。

## 2. 任务路由

- 理解项目结构 -> `架构/项目总览.md`
- 判断是否已有能力可复用 -> `架构/系统能力清单.md` + `任务导航/任务入口索引.md`
- 排查 401 / token 无效 / 网关签名错误 -> `安全/鉴权链路.md`
- 排查无权限 / 角色不匹配 -> `安全/鉴权链路.md` + `任务导航/任务入口索引.md`
- 功能开发 / 重构 -> `任务导航/任务入口索引.md` + `架构/开发约定.md`
- `UserContextUtils` 与 `SecurityContext` 使用边界 -> `安全/鉴权链路.md`
- 数据库初始化 / Nacos 配置 / 发布脚本问题 -> `运行运维/环境与运维资源.md` + `任务导航/任务入口索引.md`

## 3. 快速一致性检查

- 角色写入 `GrantedAuthority` 时必须保留 `ROLE_` 前缀。
- 权限编码为非 `ROLE_` 的 authority。
- `SecurityContext` 不存完整用户资料对象。
- 请求线程身份信息优先从网关透传请求头获取。

## 4. 更新策略

- 将新的高频问题补充到「2. 任务路由」。
- 鉴权链路变更后至少同步更新：
  - `安全/鉴权链路.md`
  - `任务导航/任务入口索引.md`
- 保持索引简洁、结构化、可检索。
