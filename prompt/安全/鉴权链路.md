# 鉴权与权限链路

## 1. 网关鉴权（WebFlux）

主入口：`mms-gateway-bc/.../filter/JwtAuthFilter.java`

执行流程：

1. 白名单路径直接放行。
2. 从 `Authorization`（或 WebSocket 子协议）提取 Bearer token。
3. 校验 access token 与黑名单状态。
4. 提取 claims：`userId`、`username`、`jti`、`exp`。
5. 生成 `X-Gateway-Signature` 与 `X-Gateway-Timestamp`。
6. 透传身份和签名请求头到下游。

## 2. 服务侧验签（Servlet）

主入口：

- `mms-common-bc-security-servlet/.../GatewaySignatureVerificationService.java`
- `mms-common-bc-security-servlet/.../JwtAuthenticationFilter.java`

执行流程：

1. 非白名单请求必须通过网关签名验证。
2. 缺少必要请求头、时间戳超窗、签名不匹配 -> `INVALID_TOKEN`。
3. 验签通过后读取透传身份，按用户名加载 authorities。
4. 构建 `UsernamePasswordAuthenticationToken` 并写入 `SecurityContext`。

## 3. 权限来源与格式

主入口：

- `mms-usercenter-bc-service/.../UserAuthorityServiceImpl.java`

格式规则：

- 角色必须写入 `ROLE_*` 形式的 authority。
- 权限编码保持原值（非 `ROLE_`）。

## 4. 方法级权限校验

主入口：

- `mms-common-bc-security-servlet/.../PermissionCheckAspect.java`
- `@RequiresPermission("permission:code")`

行为：

- 从 `SecurityContext` 读取 authorities。
- 缺少目标权限时抛出 `NO_PERMISSION`。

## 5. 身份访问规则

- 请求线程身份：`UserContextUtils.getUserId()/getUsername()`。
- 授权视图：`SecurityContext` authorities（`UserUtils.getRoles()/getPermissions()` 模式）。
- 非请求线程（异步/定时任务/MQ）必须显式传递身份参数。
