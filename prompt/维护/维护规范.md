# Prompt 知识库维护规范

## 1. 目标范围

本目录是面向编码代理的机器可检索上下文层。  
内容目标：提高检索速度、上下文一致性、执行准确性。

## 2. 编写规则

- 使用简洁、确定性的陈述。
- 避免对话语气和面向特定读者的措辞。
- 保持稳定的章节命名与可预测结构。
- 代码路径引用必须明确且保持最新。

## 3. 文件职责

- 稳定架构事实 -> `架构/项目总览.md`
- 认证/授权运行链路 -> `安全/鉴权链路.md`
- 实现约束 -> `架构/开发约定.md`
- 任务到代码映射 -> `任务导航/任务入口索引.md`
- 环境配置与运维资源 -> `运行运维/环境与运维资源.md`
- 路由与读取协议 -> `索引.md`

## 4. 更新触发条件

- 出现新的高频排障模式。
- 认证链路或请求头契约变更。
- 角色/权限格式规则变更。
- `mysql` 表结构初始化脚本变更。
- `nacos` 配置模板或服务端口/路由策略变更。
- `script` 发布/回滚/日志脚本变更。
- 模块职责变化或包路径迁移。

## 5. 一致性策略

- 代码行为是最终事实来源。
- 文档与代码不一致时，立即更新文档。
- 示例保持最小化，避免复制大段代码。
