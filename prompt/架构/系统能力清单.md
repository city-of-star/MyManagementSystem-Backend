# 系统能力清单（避免重复造轮子）

## 1. 使用规则（必须优先检查）

- 新需求落地前，先检查本清单是否已有可复用能力。
- 若已有通用能力，优先复用，避免在业务模块重复实现。
- 若能力不足，优先在原能力上扩展，再考虑新建组件。

## 2. `mms-common-bc` 通用能力

## 2.1 核心基础能力

- `mms-common-bc-core`：统一常量、上下文、异常、响应模型、通用工具。
- `mms-common-bc-webmvc`：Web MVC 自动装配、全局 Web 侧通用能力、`UserContextUtils`。
- `mms-common-bc-security-core` + `mms-common-bc-security-servlet`：
  - JWT 工具、网关签名校验、通用认证过滤器、权限注解与切面。
- `mms-common-bc-cache`：缓存相关通用封装。
- `mms-common-bc-datasource`：数据源与 MyBatis Plus 通用配置。

## 2.2 文档/文件/导出能力

- `mms-common-bc-document`：
  - `ExcelExportService`（EasyExcel 导出字节流能力）。
- `mms-common-bc-webmvc`：
  - `FileDownloadService`（统一附件/Excel 下载响应写出）。

## 2.3 线程池/任务/通信能力

- `mms-common-bc-threadpool`：线程池属性、Builder、自动装配。
- `mms-common-bc-job`：任务相关通用注解与约定（如 `JobDefinition`）。
- `mms-common-bc-websocket`：WebSocket 自动装配、握手鉴权、消息处理、会话注册、推送服务。
- `mms-common-bc-mq`：消息队列通用能力（按模块依赖使用）。

## 3. `mms-base-bc` 业务能力

## 3.1 附件管理（已具备）

- 控制器：`mms-base-bc-controller/.../AttachmentController.java`
- 能力覆盖：
  - 附件分页、详情、创建、更新、软删、硬删、批量删、状态切换。
  - 文件上传（`/attachment/upload`）。
  - 文件流式读取（`/attachment/stream/**`，用于预览/下载）。

## 3.2 系统配置（已具备）

- 控制器：`mms-base-bc-controller/.../ConfigController.java`
- 能力覆盖：配置分页、详情、按 key 查询、增改删、批量删、状态切换、key 唯一性检查。

## 3.3 数据字典（已具备）

- 控制器：
  - `.../DictTypeController.java`
  - `.../DictDataController.java`
- 能力覆盖：
  - 字典类型与字典数据全套 CRUD、分页、批量删除、状态切换。
  - 按字典类型编码查询启用字典数据（前端下拉等场景）。

## 4. `mms-job-bc` 作业能力

## 4.1 任务定义管理（已具备）

- 控制器：`mms-job-bc-core/.../JobController.java`
- 能力覆盖：任务分页、详情、增改删、批量删、状态切换、立即执行。

## 4.2 任务执行记录（已具备）

- 控制器：`mms-job-bc-core/.../JobRunLogController.java`
- 能力覆盖：执行记录分页、详情、删除、批量删除。
- 预留能力：导出、重试、终止（接口已存在，标注开发中）。

## 4.3 调度机制（已具备）

- 入口：`mms-job-bc-core/.../JobScheduler.java`
- 机制：
  - `@Scheduled(fixedDelay = 1000)` 每秒扫描到点任务。
  - 基于 `CronExpression` 计算下一次触发时间。
  - 通过执行服务异步提交任务。

## 5. 快速决策模板（实现前执行）

- 这是“通用基础能力”吗？
  - 是 -> 优先放 `mms-common-bc` 或复用既有能力。
- 这是“基础业务能力（字典/配置/附件）”吗？
  - 是 -> 优先复用 `mms-base-bc` 现有接口与服务。
- 这是“调度/执行记录”吗？
  - 是 -> 优先复用 `mms-job-bc` 任务定义与运行日志链路。
