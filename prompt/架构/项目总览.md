# 项目总览（MyManagementSystem-Backend）

## 1. 技术基线

- Java 17
- Spring Boot 3.2.4
- Spring Cloud 2023.0.1
- Spring Cloud Alibaba 2023.0.1.0
- MyBatis Plus 3.5.7

## 2. 顶层模块

- `mms-common-bc`：公共能力（`core`、`webmvc`、`security`、`cache`、`datasource` 等）。
- `mms-gateway-bc`：网关服务，负责 JWT 校验、签名生成、请求头透传。
- `mms-usercenter-bc`：身份与用户中心，负责 RBAC 与权限源数据。
- `mms-base-bc`：基础领域服务。
- `mms-job-bc`：作业/任务服务。

## 2.1 根目录资源（运行与运维）

- `mysql`：数据库初始化脚本（`mms_dev_core`/`mms_prod_core`）。
- `nacos`：配置中心示例模板（`*-DEV.yaml`），用于服务配置装配参考。
- `script`：前后端发布、回滚、重启、日志查看脚本。

## 3. 典型分层模式

- `*-common`：实体/DTO/VO/常量。
- `*-service`：业务逻辑、mapper、服务工具。
- `*-controller`：HTTP API 层。
- `*-server`：启动与服务级装配。
- `*-feign-api`：服务间调用契约。

## 4. 安全架构摘要

- 网关负责 JWT 与 token 状态校验。
- 网关透传并签名身份请求头（`userId`、`username`、`tokenJti` 等）。
- 网关使用 RSA 私钥签名；下游服务使用 RSA 公钥验签。
- 下游服务按用户名加载 authority 并写入 `SecurityContext`。

## 5. 身份读取规则

- 请求线程身份优先使用 `UserContextUtils`（来源：网关透传头）。
- 授权判断优先使用 `SecurityContextHolder` 中 authorities。
- 完整用户资料按需查询，不写入 `SecurityContext`。
